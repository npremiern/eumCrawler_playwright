## 개요

### 목적

이 문서는 eumCrawler Desktop v2.0의 모든 사용자 인터페이스 화면을 상세히 정의합니다. Svelte 5 + Tailwind CSS를 사용하여 현대적이고 반응형의 데스크톱 애플리케이션을 구현하기 위한 완전한 가이드입니다.

### 특징

- **현대적 디자인**: 깔끔하고 전문적인 UI
- **다크/라이트 모드**: 시스템 설정 따르기 지원
- **반응형 레이아웃**: 다양한 창 크기에 대응
- **접근성**: 키보드 네비게이션, 스크린 리더 지원
- **성능**: 한국어 최적화 폰트(Pretendard), 미리 로드 전략

### 대상 윈도우 크기

- **최소**: 1024 x 768px
- **권장**: 1400 x 900px 이상
- **최대**: 제약 없음 (반응형)

### Sidebar 디자인

**특징**:
- 고정 폭: 220px (또는 60px 접을 때)
- 배경: Surface color
- 드래그 가능한 타이틀바 통합
- 아이콘 + 라벨 표시

**메뉴 항목**:
1. **대시보드** (`/`)
   - 아이콘: `Home`

2. **주소 관리** (`/addresses`)
   - 아이콘: `MapPin`

3. **크롤링** (`/jobs`)

   - 아이콘: `Download`

6. **설정** (`/settings`)
   - 아이콘: `Settings`
   - Sidebar 하단 배치

#### 1.2 컴포넌트 분석

**통계 카드** (Stat Card)
- 컴포넌트: `StatCard.svelte`
- Props:
  - `title: string` - 카드 제목
  - `value: number` - 주요 수치
  - `subtitle: string` - 부제목 (예: "+12 today")
  - `icon: string` - 아이콘 ID
  - `color: 'blue' | 'green' | 'amber' | 'violet'` - 색상

**빠른 작업 버튼**
- 컴포넌트: `QuickActionButton.svelte`
- 스타일: 전폭 버튼, 좌측 아이콘, 우측 텍스트
- 호버: 배경색 변경, 약간의 scale-up

**최근 작업 카드**
- 컴포넌트: `JobHistoryCard.svelte`
- Props:
  - `job: Job` - 작업 객체
  - `onDetail: () => void` - 상세보기 클릭
  - `onRetry: () => void` - 재실행 클릭

**결과 테이블**
- 컴포넌트: `ResultTable.svelte`
- Props:
  - `results: Result[]`
  - `onRowClick: (id: number) => void`
- 스타일: 스트라이프 행, 호버 하이라이트

#### 1.3 상태 관리

**Svelte Stores**:
```typescript
// stores/dashboard.ts
export const dashboardStats = $state({
  totalAddresses: 1234,
  completedAddresses: 1100,
  successRate: 0.891,
  cacheCount: 89,
  cacheTotalSize: 45, // MB
  cacheExpiringCount: 3
});

export const recentJobs = $state<Job[]>([]);
export const recentResults = $state<Result[]>([]);
export const isLoading = $state(false);
```

**로드 함수**:
```typescript
async function loadDashboard() {
  isLoading = true;
  try {
    dashboardStats = await invoke('get_dashboard_stats');
    recentJobs = await invoke('get_recent_jobs', { limit: 5 });
    recentResults = await invoke('get_recent_results', { limit: 10 });
  } finally {
    isLoading = false;
  }
}
```

#### 2.2 컴포넌트 분석

**액션 바**:
- [+ 주소 추가]: AddressAddModal 열기
- [📁 Excel 가져오기]: AddressImportModal 열기
- [🗑 선택 삭제]: 확인 다이얼로그 후 벌크 삭제

**검색 & 필터**:
- 검색어 입력: 실시간 필터링 (debounce 300ms)
- 그룹 필터: 드롭다운 (전체, 서울, 경기, 부산, ...)
- 태그 필터: 멀티 셀렉트 (선택사항)

**주소 테이블**:
- 컬럼: 체크박스, ID, 주소, 그룹, 태그
- 행 호버: 배경색 변경 (gray-50)
- 행 클릭: 주소 상세 정보 모달 (선택사항)

### Base UI Components

#### Button.svelte
```svelte
<Button
  variant="primary|secondary|outline|ghost"
  size="sm|md|lg"
  disabled={false}
  loading={false}
  on:click
>
  Click me
</Button>
```

Props:
- `variant`: 버튼 스타일 (primary=파란색, secondary=회색, outline=테두리, ghost=투명)
- `size`: 크기 (sm=작음, md=중간, lg=큼)
- `disabled`: 비활성 상태
- `loading`: 로딩 스피너 표시
- `icon`: (선택) 좌측 아이콘 ID

#### Input.svelte
```svelte
<Input
  type="text|number|email|password"
  placeholder="Enter value"
  value=""
  on:change
  on:input
  error={null}
  disabled={false}
/>
```

Props:
- `type`: 입력 타입
- `placeholder`: 플레이스홀더 텍스트
- `value`: 입력값
- `error`: 에러 메시지 (표시 시 빨간색 테두리 + 에러 텍스트)
- `disabled`: 비활성 상태

#### Modal.svelte
```svelte
<Modal title="Modal Title" isOpen={true} on:close>
  <slot />

  <svelte:fragment slot="footer">
    <Button variant="secondary">Cancel</Button>
    <Button variant="primary">Confirm</Button>
  </svelte:fragment>
</Modal>
```

Props:
- `title`: 모달 제목
- `isOpen`: 표시 여부
- `size`: "sm|md|lg" (기본: md)
- Slots: 기본 콘텐츠, footer (버튼)

#### Table.svelte
```svelte
<Table columns={[...]} rows={[...]} on:rowClick>
  <svelte:fragment slot="cell-custom" let:row>
    <!-- 커스텀 셀 렌더링 -->
  </svelte:fragment>
</Table>
```

Props:
- `columns`: 컬럼 정의 배열
- `rows`: 행 데이터 배열
- `selectable`: 체크박스 포함 여부
- `selectedRows`: 선택된 행 배열

#### Toast.svelte
```svelte
<Toast type="success|warning|error|info" message="Message" duration={3000} />
```

Props:
- `type`: 토스트 유형
- `message`: 메시지
- `duration`: 자동 닫기 시간 (ms)
- `action`: (선택) 액션 버튼

#### ProgressBar.svelte
```svelte
<ProgressBar value={45} max={100} label="45%" animated />
```

Props:
- `value`: 현재 값
- `max`: 최대값
- `label`: 표시 텍스트
- `animated`: 애니메이션 여부
- `color`: "blue|green|amber|red"

## 상태 관리

### Svelte Stores Architecture

모든 stores는 다음 패턴을 따릅니다:

```typescript
// stores/example.ts
import { invoke } from '@tauri-apps/api/core';

// 상태 정의
let items = $state<Item[]>([]);
let isLoading = $state(false);
let error = $state<string | null>(null);
let pagination = $state({ page: 1, size: 50 });

// 파생 상태
let filteredItems = $derived(
  items.filter(item => item.status === 'active')
);
let totalPages = $derived(Math.ceil(items.length / pagination.size));

// 효과 (side effects)
$effect(() => {
  if (error) {
    const timer = setTimeout(() => { error = null; }, 5000);
    return () => clearTimeout(timer);
  }
});

// 액션 함수들
export async function loadItems() {
  isLoading = true;
  error = null;
  try {
    items = await invoke('get_items');
  } catch (err) {
    error = String(err);
  } finally {
    isLoading = false;
  }
}

// Export store 객체
export function getItemsStore() {
  return {
    get items() { return items; },
    get isLoading() { return isLoading; },
    get error() { return error; },
    get filteredItems() { return filteredItems; },
    loadItems,
    // ... 기타 액션
  };
}
```

### 주요 Stores

1. **addressStore**: 주소 관리
2. **jobStore**: 크롤링 작업
3. **resultStore**: 조회 결과
4. **settingsStore**: 설정
5. **uiStore**: UI 상태 (모달, 토스트)

```typescript
onMount(async () => {
  try {
    // 1. 설정 로드 (빠름, 필수)
    await settingsStore.loadSettings();

    // 2. 메인 데이터 로드 (병렬)
```
